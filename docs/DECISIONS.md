# Журнал архитектурных решений (ADR)

Этот документ фиксирует **почему** были приняты те или иные решения.
Помогает новым разработчикам (и вам через полгода) понять контекст.

---

```

**Дата:** YYYY-MM-DD

**Последствия:** Что это значит
```

## ADR-001: Монорепозиторий для всех компонентов

**Дата:** 2026-04-04
**Статус:** Принято
**Контекст:** Проект включает бэкенд (FastAPI), веб-фронтенд (React), десктоп (Electron), мобильный (React Native/Flutter). Нужна стратегия организации кода.

**Решение:** Монорепозиторий с папками `server/`, `sidebar/`, `desktop/`, `mobile/`, `shared/`.

**Последствия:**
- Одна история Git, один `git pull`
- При изменении API сразу обновляются все клиенты
- Репозиторий будет расти — при >5 GB разделим

## ADR-002: Абстрактный слой БД (SQLite + PostgreSQL)

**Дата:** 2026-04-04
**Статус:** Принято
**Контекст:** Локальная разработка на SQLite (файл), сервер — на PostgreSQL. Нужен один код репозиториев.

**Решение:** Создан `db/connection.py` с абстрактными `Connection`, `Cursor`, `Row`. Переключение через `DATABASE_TYPE` env var.

**Последствия:**
- Все репозитории работают через абстрактный `Connection`
- SQL одинаковый, `?` автоматически → `%s` для PostgreSQL
- Миграции (`ALTER TABLE`) только для SQLite

## ADR-003: FastAPI раздаёт статику

**Дата:** 2026-04-04
**Статус:** Принято
**Контекст:** Фронтенд (React) и API (FastAPI) должны работать на одном порту для простоты развёртывания.

**Решение:** FastAPI через `StaticFiles` раздаёт `sidebar/build/`. API роуты (`/api/*`) обрабатываются до статики.

**Последствия:**
- Один порт (8000) — проще Docker, Nginx, клиенты
- `API_BASE_URL = ''` во фронтенде (относительные пути)
- При разработке — раздельные серверы через `REACT_APP_API_URL`

## ADR-004: PyInstaller для Windows-EXE

**Дата:** 2026-04-04
**Статус:** Принято
**Контекст:** Нужен вариант "нажал кнопку — работает" на Windows без установки Python.

**Решение:** Multi-stage PyInstaller сборка. Фронтенд встраивается в EXE. Статика ищется через `sys._MEIPASS`.

**Последствия:**
- `dist/MyInfoManager/` — готовый к распространению
- Python не нужен на целевой машине
- Размер ~100-200 MB

## ADR-005: JWT для авторизации

**Дата:** 2026-04-04
**Статус:** Принято
**Контекст:** Несколько клиентов, каждый видит только свои данные. Нужна аутентификация без сессий (мобильные, API).

**Решение:** JWT (JSON Web Tokens) с access/refresh токенами. Библиотеки: `python-jose`, `passlib[bcrypt]`.

**Последствия:**
- Stateless — сервер не хранит сессии
- Refresh-токен для долгого входа
- Все CRUD операции фильтруются по `user_id`
- HTTPS обязателен для production

## ADR-006: `user_id` во всех таблицах данных

**Дата:** 2026-04-04
**Статус:** Принято
**Контекст:** Разделение данных между пользователями.

**Решение:** Добавить `user_id INTEGER REFERENCES "user"(id) ON DELETE CASCADE` во все таблицы данных.

**Последствия:**
- При удалении пользователя — все его данные удаляются
- Репозитории фильтруют по `user_id`
- Системные таблицы (`note_tag`, `subtask`, `settings`) без `user_id`

## ADR-007: Docker Compose для сервера

**Дата:** 2026-04-04
**Статус:** Принято
**Контекст:** Развёртывание на VPS. Несколько сервис, общая БД, Nginx как reverse proxy.

**Решение:** Два docker-compose: `infra/` (Nginx + PostgreSQL) и корневой (MyInfoManager). Общая сеть через внешний DNS или IP.

**Последствия:**
- Инфраструктура обновляется отдельно
- Сервисы независимы
- Для production — SSL через Let's Encrypt

## ADR-008: Самоподписанный SSL для тестов

**Дата:** 2026-04-04
**Статус:** Принято
**Контекст:** Локальная VM без внешнего IP. Let's Encrypt требует доступ из интернета.

**Решение:** `openssl req -x509` для генерации самоподписанного сертификата. Браузер предупреждает — принимаем вручную.

**Последствия:**
- HTTPS работает в локальной сети
- Мобильные клиенты потребуют доверия к сертификату
- При переходе на VPS — замена на Let's Encrypt

## ADR-009: Nginx на порту 80 вместо Apache (Nextcloud)

**Дата:** 2026-04-04
**Статус:** Принято
**Контекст:** На VM установлен Nextcloud (snap), который занимает порт 80.

**Решение:** `sudo snap stop nextcloud` для тестов. В будущем — Nginx на 443, Nextcloud на другом порту или отдельная VM.

**Последствия:**
- Nextcloud временно недоступен
- При включении — конфликт портов
