## Ресурсы

### ✅ Список ресурсов

**Описание:** Просмотр всех информационных ресурсов с разбивкой по категориям.

**Функции:**
- Отображение в виде дерева (категории → ресурсы)
- Счётчик ресурсов в категории
- Быстрый переход по URL

**UI:**
- Sidebar → иконка "📁" → панель "Ресурсы"
- Клик по категории → раскрыть/свернуть
- Клик по ресурсу → открыть в основной панели

---

### ✅ Создание ресурса

**Описание:** Добавление нового информационного ресурса.

**Поля:**
- Название (обязательно)
- Категория (опционально)
- URL (опционально)
- Описание (опционально)

**UI:**
- Кнопка "+ Ресурс" в панели ресурсов
- Форма с полями
- Кнопка "OK" для сохранения

---

### ✅ Редактирование ресурса

**Описание:** Изменение существующего ресурса.

**Функции:**
- Изменение всех полей
- Сохранение изменений
- Отмена редактирования

**UI:**
- Клик по ресурсу → основная панель
- Кнопка "✏️ Редактировать"
- Форма с текущими значениями
- Кнопки "Сохранить" / "Отмена"

---

### ✅ Удаление ресурса

**Описание:** Удаление ресурса из базы данных.

**Функции:**
- Подтверждение удаления
- Удаление из списка

**UI:**
- Кнопка "🗑️" рядом с ресурсом
- Диалог подтверждения

---

### ✅ Поиск ресурсов

**Описание:** Поиск ресурсов по названию.

**Функции:**
- Поиск по частичному совпадению
- Поиск без учёта регистра
- Отображение результатов

**UI:**
- Поле поиска в панели ресурсов
- Автоматический поиск при вводе (debounce 300ms)

---

### ✅ Категории ресурсов

**Описание:** Управление категориями ресурсов.

**Функции:**
- Создание категории
- Удаление категории

**UI:**
- Кнопка "+ Категория"
- Кнопка "🗑️" рядом с категорией

- Аудио-плеер

## Синхронизация

### ✅ Модуль синхронизации

**Описание:** Синхронизация данных между локальной БД и удалённым сервером.

**Функции:**
- Сравнение версий по timestamp
- Обнаружение конфликтов
- Разрешение конфликтов
- Слияние данных

**Стратегии разрешения конфликтов:**
- `local_wins` - локальная версия побеждает
- `remote_wins` - удалённая версия побеждает
- `newer_wins` - более новая версия побеждает
- `manual` - ручное разрешение

**Типы конфликтов:**
- `modified_both` - изменено в обоих местах
- `deleted_on_remote` - удалено на сервере
- `deleted_locally` - удалено локально

**Статус:** ⚠️ Модуль готов, требуется удалённый сервер
