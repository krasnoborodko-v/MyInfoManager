## 📊 Ключевые решения

### Архитектурные решения

1. **Вложения в БД** - файлы хранятся в SQLite (BLOB)
   - ✅ Простое резервное копирование
   - ❌ БД может стать большой

2. **Теги** - отношение многие-ко-многим
   - Таблица `note_tag` для связи
   - Одна заметка → много тегов
   - Один тег → много заметок

3. **Папки** - иерархическая структура
   - Рекурсивное отображение
   - Связь с категориями заметок

4. **Синхронизация** - модуль готов
   - 4 стратегии разрешения конфликтов
   - Требуется удалённый сервер

### UI/UX решения

1. **Компактный интерфейс** - экономия места
   - Заголовки + кнопки в одной строке
   - Категория + дата в одной строке
   - Теги + кнопка в одной строке

2. **Контекстное меню** - правый клик
   - Перемещение заметок между папками
   - Быстрый доступ к действиям

3. **Цветовая схема**
   - Фон: `#001616`
   - Текст: `#aaffff`
   - Акценты: `#008888`

## 🐛 Известные ограничения

### Производительность

- При 1000+ заметок может замедляться поиск
- Рекомендуется добавить индексы на большие таблицы

### Вложения

- Файлы >10MB не загружаются (ограничение API)
- При большом количестве файлов БД растёт

### Синхронизация

- Модуль готов, но сервер не реализован
- Локальная БД работает независимо
